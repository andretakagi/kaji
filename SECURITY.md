# Security Policy

## Reporting a Vulnerability

If you find a security issue in Kaji, please report it privately using
[GitHub's vulnerability reporting](https://github.com/andretakagi/kaji/security/advisories/new)
rather than opening a public issue.

## What to expect

Kaji is a solo-maintained project. I take security seriously but can't guarantee
specific response times. I'll acknowledge reports as soon as I can and work on
fixes as my schedule allows.

## Scope

This policy covers the Kaji application itself. For issues in Caddy, please
report those to the [Caddy project](https://github.com/caddyserver/caddy) directly.
